# Contributing

Merci de votre intérêt pour contribuer à ce projet.

## Procédure recommandée

- Forkez le dépôt et créez une branche feature/xxx.
- Travaillez sur des changements atomiques et poussez sur votre fork.
- Ouvrez une Pull Request claire décrivant les changements et la motivation.

## Style de code

- Java 11+.
- Javadoc pour toutes les classes publiques et méthodes importantes.
- Préférez des noms explicites et des méthodes courtes.

## Tests

- Fournissez des petits tests ou des scénarios d'exécution lorsque c'est pertinent.
- Le CI compile l'ensemble du projet et exécute un petit test smoke.

## Commit messages

- Utilisez des messages clairs et impératifs, ex : `Ajouter sortie CSV du tournoi`.
- Si la PR corrige un ticket, mentionnez le numéro.

## Licence

- Le dépôt contient une licence `LICENSE` (MIT par défaut). Assurez-vous que vos contributions
  sont compatibles avec cette licence.
